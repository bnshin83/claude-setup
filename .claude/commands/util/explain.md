# Explain

Simple, intuition-first explanations.

## Usage
```
/util/explain [concept or code]
```

## Examples
```
/util/explain attention mechanism
/util/explain this function
/util/explain "what does this regex do"
```

## Approach
1. One-sentence summary
2. Core intuition (analogy if helpful)
3. Key details (only if needed)
4. Example (concrete, minimal)

## Output Style
- No jargon without definition
- Shortest explanation that works
- Concrete over abstract
- Examples over theory
